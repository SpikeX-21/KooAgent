## 测试目标

提示要求模型只使用 Android Extension 工具：

1. `android_list_installed_apps`
2. `android_query_memory({ query: "Operit" })`
3. `android_use_package({ package_name: "__kimi_k25_missing_package__" })`
4. 读取第 3 步的模型可见错误摘要；若 `retryable=false` 且 `userActionRequired=true`，禁止重试 package，改为 `android_sleep({ duration_ms: 20 })`。

这覆盖了：LLM 工具选择、Pi tool loop、Extension 请求/结果映射、Operit v2 执行、Android executor、失败结果进入下一轮 LLM 上下文，以及恢复决策。

第 3 步进入 Pi `toolResult.content` 的第一个文本块为：

```text

status=FAILED
code=PACKAGE_ACTIVATION_FAILED
category=PRECONDITION
retryable=false
userActionRequired=true
message="Package not found: __kimi_k25_missing_package__. Please import it first or register it as an MCP server."
```

同一 `toolResult` 还保留原始错误文本；完整 `RemoteToolOutcomeV2` 位于 `details.outcome`，并带有 trace、timing、runtime 与结构化 error。Pi 将该工具结果标记为 `isError: true`。

## LLM 恢复决策证据

Kimi K2.5 的下一轮 reasoning 明确提取：

```text
status=FAILED
code=PACKAGE_ACTIVATION_FAILED
category=PRECONDITION
retryable=false
userActionRequired=true
```

随后它选择：不重试 `android_use_package`，调用 `android_sleep({ duration_ms: 20 })`。最终回答也准确说明：错误不可重试且需要用户先导入 package 或注册 MCP server。

这证明结构化失败摘要不仅被写入 Pi session 的 `content`，还实际被默认模型 Kimi K2.5 消费并影响了后续工具调用。
